# Slot 消除機制與馬可夫鏈的關係
## — 從體感設計看背後的數學模型

---

## 文章大綱

### 一、前言：消除類 Slot 的崛起
- 近年 cascade/tumble 機制越來越流行（例：Sweet Bonanza、Gates of Olympus、Sugar Rush）
- 核心玩法：消除 → 掉落新 symbol → 再消除 → 累積/收集 → 觸發 Feature
- 玩家的「快要中了」的興奮感，其實是精心設計的數學模型

### 二、什麼是馬可夫鏈（Markov Chain）？
> 用最白話的方式解釋，不需要數學背景也能懂

**定義：** 下一個狀態的機率，只取決於「目前的狀態」，跟之前的歷史無關。

**生活類比：**
- 天氣預報：今天晴天 → 明天晴天 70%、陰天 20%、雨天 10%
- 只看「今天」，不管昨天前天

**圖示（狀態轉移圖）：**
```
        0.7
    ┌────────┐
    ↓        │
[晴天] ──0.2──→ [陰天]
    │              │
   0.1            0.4
    ↓              ↓
  [雨天] ←──0.3──┘
```

### 三、消除類 Slot 的運作流程
> 圖解：一次 Spin 的完整生命週期

```
Spin 開始
    ↓
初始盤面（State 0）
    ↓
第 1 次消除？ ──No──→ 本輪結束（無連消）
    ↓ Yes
消除 + 賠付（State 1）
    ↓
新 symbol 掉落
    ↓
第 2 次消除？ ──No──→ 本輪結束（1 次連消）
    ↓ Yes
消除 + 賠付（State 2）
    ↓
新 symbol 掉落
    ↓
...持續到無法消除為止
    ↓
累積達標？ ──Yes──→ 觸發 Feature！
    ↓ No
本輪結束
```

### 四、關鍵洞察：每一次消除的「狀態」不同

這是文章的核心論點。

**觀察現象：**

| 連消次數 | 常見消除pattern | 體感 | 設計意圖 |
|---------|----------------|------|---------|
| 第 1 消 | 基本符號 3-4 個（前 3 軸居多） | 「喔，有消」 | 低賠付，開啟連消可能 |
| 第 2 消 | 基本符號 3-5 個 | 「不錯，繼續」 | 維持期待感 |
| 第 3 消 | 基本/中階符號 | 「要來了嗎？」 | 接近 Feature 門檻 |
| 第 4 消 | 可能出現高階符號 | 「快了快了！」 | 賠付可能增大 |
| 第 5+ 消 | 大消除或觸發 Feature | 「中了！」 | 高潮或落空 |

**體感關鍵：** 前幾次消除故意「小消」，每次只消 3-4 個基本符號，讓你：
1. 感覺「有在連」→ 多巴胺微量釋放
2. 賠付很小 → 遊戲方控制 RTP
3. 累積收集進度 → 接近 Feature 門檻
4. 製造「差一點」的錯覺

### 五、馬可夫鏈建模

> 把每次消除後的盤面狀態，定義為馬可夫鏈的「狀態」

**狀態定義：**
- **S0**：初始盤面（剛 spin 完）
- **S1**：第 1 次消除後
- **S2**：第 2 次消除後
- **S3**：第 3 次消除後
- **S4+**：第 4 次以上消除
- **F**：觸發 Feature（吸收態）
- **E**：結束（吸收態）

**轉移機率矩陣（範例）：**

|  | S0 | S1 | S2 | S3 | S4+ | F | E |
|--|----|----|----|----|-----|---|---|
| **S0** | 0 | 0.35 | 0 | 0 | 0 | 0 | 0.65 |
| **S1** | 0 | 0 | 0.40 | 0 | 0 | 0 | 0.60 |
| **S2** | 0 | 0 | 0 | 0.35 | 0 | 0.02 | 0.63 |
| **S3** | 0 | 0 | 0 | 0 | 0.30 | 0.05 | 0.65 |
| **S4+** | 0 | 0 | 0 | 0 | 0.25 | 0.10 | 0.65 |
| **F** | 0 | 0 | 0 | 0 | 0 | 1 | 0 |
| **E** | 0 | 0 | 0 | 0 | 0 | 0 | 1 |

**解讀：**
- 從 S0（初始）有 35% 機率發生第一次消除，65% 直接結束
- 每多一次連消，繼續消的機率略微降低（40% → 35% → 30% → 25%）
- 但 Feature 觸發機率逐漸升高（0% → 2% → 5% → 10%）
- 這就是為什麼「連消越多次越興奮」— 數學上你確實更接近 Feature

### 六、不同遊戲的馬可夫模型比較

#### 類型 A：收集型（如 Sweet Bonanza 的棒棒糖 scatter）
> 收集 4/5/6 個 scatter 觸發 Feature

```
狀態 = scatter 收集數量
S0(0個) → S1(1個) → S2(2個) → S3(3個) → Feature(4+個)
```

**特點：** 每次消除都有固定機率出現 scatter，收集是累加的

#### 類型 B：連消計數型（如 Reactoonz 的連消觸發）
> 連消 N 次觸發 Feature

```
狀態 = 連消次數
每次連消的機率隨狀態改變
```

**特點：** 連消次數本身就是狀態，符合馬可夫性質

#### 類型 C：充能型（如 Jammin' Jars 的倍率累積）
> 消除累積倍率，倍率夠高時本身就是 Feature

```
狀態 = 當前倍率等級
低倍(×1-2) → 中倍(×3-5) → 高倍(×6-10) → 爆發(×10+)
```

**轉移矩陣對比表：**

| 遊戲類型 | 連消維持率 | Feature 觸發率/次 | 平均連消次數 | 體感 |
|---------|-----------|------------------|------------|------|
| 收集型 | 高 (40-50%) | 低 (~1-2%) | 2-3 次 | 慢慢接近 |
| 連消計數型 | 中 (30-40%) | 中 (~3-5%) | 1-2 次 | 爆發或落空 |
| 充能型 | 高 (45-55%) | 漸進式 | 3-5 次 | 越打越大 |

### 七、體感設計的數學秘密

**秘密 1：前幾消的「小消」是刻意的**

第 1-2 次消除的 reel strip 設計：
- 高權重基本符號（佔 60-70%）
- 消除 cluster 大小偏小（3-4 個）
- 消除位置集中在前 3 軸（控制賠付）

```
消除規模分佈（以第 1 消為例）：

3 個符號消除: ████████████████ 55%
4 個符號消除: █████████ 25%
5 個符號消除: █████ 12%
6+ 個符號消除: ███ 8%
```

**秘密 2：「差一點」的頻率被精確控制**

假設 Feature 需要 5 次連消：
- 達到 3 次連消的機率 ≈ 5-8%（讓你覺得「快了」）
- 達到 4 次連消的機率 ≈ 2-3%（「就差一點！」）
- 真正觸發 Feature ≈ 0.5-1%（控制 RTP）

```
連消深度分佈圖：

0 次（沒消）: ████████████████████████████ 65%
1 次連消:     ██████████ 20%
2 次連消:     ████ 8%
3 次連消:     ██ 4%
4 次連消:     █ 2%
5+ 次（Feature）: ▎ 1%
```

**這就是為什麼你總覺得「差一點就中了」— 遊戲設計者精確控制了這個「差一點」出現的頻率，讓它剛好夠多到維持你的期待，但又不會多到真的常常觸發。**

### 八、實際計算範例

**假設一款遊戲的轉移矩陣如下：**

從任意 spin 開始，計算觸發 Feature 的總機率：

```
P(Feature) = P(S0→S1) × P(S1→S2) × P(S2→S3) × P(S3→S4) × P(S4→F)
           + P(S0→S1) × P(S1→S2) × P(S2→F)
           + ...（所有可能路徑加總）

數值範例：
= 0.35 × 0.40 × 0.35 × 0.30 × 0.10  (5次連消路徑)
+ 0.35 × 0.40 × 0.35 × 0.05          (4次連消路徑)
+ 0.35 × 0.40 × 0.02                  (3次連消路徑)
= 0.00147 + 0.00245 + 0.0028
= 0.00672
≈ 0.67%

即每 ~149 次 spin 觸發一次 Feature
```

### 九、對玩家的啟示

1. **每一次消除的「感覺」是被設計的** — 小消不代表接下來不會大消，但機率確實在變化
2. **「差一點」是最強的心理陷阱** — 3-4 次連消的頻率被刻意控制在一個讓你覺得「下一把可能中」的甜蜜點
3. **馬可夫性質意味著：過去不影響未來** — 上一把連消 4 次沒中 Feature，不代表下一把更容易中
4. **RTP 是長期保證，不是短期承諾** — 這些轉移機率被精確計算，確保長期回報率符合設定

### 十、結語

消除類 Slot 的連消機制，本質上就是一個精心設計的馬可夫鏈。每一次消除都是一次狀態轉移，而每個狀態的轉移機率都被遊戲設計者精確調整，目的是：

1. 控制 RTP（回報率）
2. 製造「接近 Feature」的體感
3. 維持玩家的參與度

理解了這個數學模型，你就能更理性地看待每一次「差一點就中了」的體驗。
